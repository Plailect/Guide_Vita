### 必讀事項

h-encore 是一系列專門用於 PS Vita (TV) 的漏洞；此漏洞將允許您安裝自製軟體至您的主畫面中。 它與 3.65 至 3.68 的系統韌體相容。

請注意，h-encore 並非永久性的漏洞 (也就是說該漏洞將於主機重啟後失效)。 For the purposes of these instructions, we will be merely using it in order to gain the level of access required to downgrade to firmware version 3.60 in order to install more powerful exploits.

除了安裝 h-encore 漏洞外，我們也將啟用『非安全』的自製程式使用權以提高自製程式的執行權限。 這類似於電腦中的「管理員」權限。

如果您使用的是 PS Vita 1000，那麼您必須要有一張原廠 Sony 記憶卡 (任意大小) 才能繼續。 此限制不適用於 PS Vita 2000 或 PS TV，因為這些主機皆有內建記憶卡/內存。
{: .notice--info}

### 必備項目

* 最新版的 [finalhe](https://github.com/soarqin/finalhe/releases/latest)

### 操作說明

#### 第一節 — finalhe

1. 解壓 finalhe `.zip` 的檔案至您電腦的一個資料夾中
1. 在您的電腦中啟動 finalhe
  + 如果您使用的是 Windows 電腦且系統跳出防火牆警示，請允許 finalhe 連接至網路
1. 於您的主機中啟動「內容管理員」
1. 選擇『複製內容』
1. 選擇『個人電腦』
1. 選擇您想要連接至 finalhe 的方式
  + 如果您被要求登入至 PlayStation Network，請登入
  + 如果您並沒有一隻 PlayStation Network 的帳號，請創立一隻
1. 如提示時，選擇／註冊您的電腦
  + 若有提示更新的警示，請重新啟動您的主機並再試一次
  + If you still encounter this message, follow the instructions on [Blocking Updates](blocking-updates) and try again
  + If you *stil* encounter this message, enable Flight Mode in the System Settings and try again (this will *not* work on PS TVs)
  + 如果您的 Windows 電腦偵測不到您主機的 USB 連線的話，請安裝 [QcmaDriver_winusb](https://github.com/soarqin/finalhe/releases/download/v1.3/QcmaDriver_winusb.exe) 並再試一次

#### 第二節 — h-encore 轉移

1. 選擇『Trim h-encore to ~13MB』
1. 選擇『Let's GO!』
  + 漏洞所需檔案將自動被下載及自動進行其前置作業
  + 此過程將花費一些時間
1. 於您的主機中選擇『個人電腦 -> PS Vita』
1. 選擇『應用程式』
1. 選擇『PS Vita』
1. 選擇『h-encore』
1. 選擇『複製』
1. 選擇『OK』
  + h-encore 漏洞將會被複製到您的主機中
  + 此過程將花費一些時間
1. 於您的主機中關閉「內容管理員」
1. 在您的電腦中關閉 finalhe

#### Section III - Configuring h-encore

1. 啟動 h-encore
1. 如有提示關於獎盃的事項，選擇「是」
1. 如果漏洞成功執行的話，您將會看到 h-encore 引導選單
  + 如果執行此漏洞時卡在白屏的畫面的話，試著關閉本程式 (過程中您的主機會當機並重啟) 並再試一次
  + 如果您仍然卡死在此步驟，按住電源紐30秒以強制重啟，並再試一次
1. 選擇『Install HENkaku』
  + 這將會安裝 HENkaku 漏洞，並啟用自製軟體存取權直到下次重啟
1. Select "Download VitaShell"
  + This will install the VitaShell homebrew application for managing your device's filesystem
  + VitaShell (and all homebrew applications in general) will remain installed after a reboot, but will give an error on launch if the HENkaku exploit is not active
1. 選擇『Exit』

#### Section IV - Configuring HENkaku

1. 啟動設定
1. 移動至 `HENkaku 設置`
  + If HENkaku Settings is missing, select the "Reset taiHEN config.txt" in the h-encore bootstrap menu, then try again
1. 勾選『啟動不安全自制軟件』
1. 回到 HENkaku 設置選單
1. 關閉設定
